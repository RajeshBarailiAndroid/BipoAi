Connect Gemini on production

Local dev uses your Mac .env file. bipoai.com uses Vercel — you must add the same secrets there and redeploy.

    Fix in 5 steps

    1. On your Mac, in the BipoAi project folder, run: npm run vercel:import-file
    2. Open vercel.com/dashboard → project for bipoai.com
    3. Settings → Environment Variables → Import .env (upload vercel-import.env)
    4. Enable Production, Preview, and Development → Save
    5. Deployments → Redeploy (required — saving alone is not enough)

    Required variables

    Use these exact names in Vercel:

    • GEMINI_API_KEY — from Google AI Studio (AIzaSy… recommended)
    • GEMINI_MODEL · GEMINI_SOLVE_MODEL
    • SUPABASE_URL · SUPABASE_SERVICE_ROLE_KEY · SUPABASE_ANON_KEY
    • SESSION_SECRET · GOOGLE_CLIENT_ID · GOOGLE_CLIENT_SECRET
